# Working Memory

Baddeley's four-component model of working memory — the system for temporarily holding and manipulating information during active cognition.

## Components

| Component | Function |
|---|---|
| **Central Executive** | Attentional control; coordinates the other components; bottleneck |
| **Phonological Loop** | Holds verbal/acoustic information; inner voice |
| **Visuospatial Sketchpad** | Holds visual and spatial information; inner eye |
| **Episodic Buffer** | Integrates information across modalities; links to long-term memory |

## Capacity Limits

The central executive can hold approximately **4 ± 1 chunks** of information simultaneously. A "chunk" is a meaningful unit — an expert's chunks are larger than a novice's (expertise is larger chunks in long-term memory).

## The Bottleneck

The central executive (prefrontal cortex-mediated) is the bottleneck of conscious thought. When it's overloaded:
- Learning fails (can't encode new information)
- Performance degrades (can't coordinate ongoing processes)
- Errors increase

## LLM Analogy

The context window of a language model is a functional analogue: fixed capacity, active manipulation of contents, bottleneck for complex reasoning. Context engineering is working memory management for AI systems.

- [[Cognitive Load Theory]] — CLT models the central executive as the capacity that intrinsic + extraneous load must not exhaust
