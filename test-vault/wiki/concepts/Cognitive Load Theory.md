# Cognitive Load Theory

Sweller (1988): learning fails not because content is hard but because the *presentation* imposes unnecessary load that exhausts working memory before the material reaches long-term memory.

## Three Load Types

| Type | Source | Strategy |
|---|---|---|
| **Intrinsic** | Inherent complexity of the material; number of interacting elements | Sequence instruction to reduce concurrent elements |
| **Extraneous** | Poor design: redundant info, confusing layout, unnecessary processing | Eliminate; use worked examples, clear formatting |
| **Germane** | Effortful processing that builds schemas | Maximize within remaining WM capacity |

## The Central Executive as Bottleneck

Intrinsic + extraneous load must not exceed the central executive's capacity (~4 chunks). When they do, germane load (the learning) can't happen.

**Reducing friction ≠ reducing learning.** Reducing *extraneous* load frees capacity for *germane* load. Making learning *feel* easier by removing challenge reduces germane load and produces fluency without schema formation.

## Expertise Reversal Effect

Worked examples (which reduce intrinsic + extraneous load) help novices but *hurt* experts. Experts' existing schemas mean the worked example adds redundant load. Scaffolding must fade as expertise grows.
