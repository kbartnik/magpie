# Neurodiverse SE Research Post-LLM Validity

*Do the 2015–2023 findings on ADHD in software engineering remain valid in the post-remote-work, post-LLM-tooling context?*

Liebel 2023 and GetMeInTheGroove (ICSE 2025) are the primary evidence bases for understanding ADHD in software engineering. Both were conducted when the standard SE environment was office-based or transitioning to hybrid, and before LLM-assisted coding tools became widespread.

The remote-work shift changed ADHD-relevant dimensions significantly: eliminated office sensory overload triggers, increased work schedule autonomy, reduced interruption-based task switching. For many ADHD-diagnosed developers, remote work was an accessibility win. Post-pandemic hybrid returns may have reversed some of these gains.

LLM-assisted coding tools may have fundamentally changed which SE tasks are most challenging for ADHD. If AI handles working memory demands (tracking complex call chains, API signatures, project context), the challenge profile shifts from memory-and-attention tasks toward the judgment and direction tasks AI can't fully handle. Whether this makes SE more or less ADHD-accessible is an empirical question that hasn't been studied.
