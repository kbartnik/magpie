## YAML parsing in Go

`gopkg.in/yaml.v3` is already in `go.mod` (pulled in transitively). Two core functions:

- `yaml.Unmarshal(data []byte, v any)` — for bytes; right choice for `Load`
- `yaml.NewDecoder(r io.Reader)` — for streaming from a reader

yaml.v3 maps YAML keys to struct fields via `yaml:` struct tags. `inbox_path` in the file → `InboxPath string \`yaml:"inbox_path"\``.

Basic pattern:

```go
data, err := os.ReadFile(path)
if err != nil {
    return nil, err
}

var cfg Config
if err := yaml.Unmarshal(data, &cfg); err != nil {
    return nil, err
}
return &cfg, nil
```

## Load contract

Three cases to handle:

**Missing file:** `os.ReadFile` returns a `*PathError` wrapping `fs.ErrNotExist`. Check with `errors.Is(err, fs.ErrNotExist)` and return an empty `&Config{}` (not an error) — that's the doc comment contract.

**Malformed YAML:** `yaml.Unmarshal` returns a `*yaml.TypeError` for type mismatches or a syntax error for unparseable input. Return as-is — no special handling needed.

**Absent keys:** fields not present in the file stay at their zero value (`""`). Correct for an all-optional config.

Full shape: read file → handle missing → unmarshal → return. ~15 lines.

## Logging at the caller, not the package

Go packages don't log — callers do. `Load` communicates outcome through return values; if the caller cares whether the config file was missing vs found, it can check and log accordingly. Adding `slog.Default()` inside `Load` would reach for global state and fire in tests.

Decision: no logging in `Load`. The cobra root command (which calls `Load`) is the right place to emit a debug message if needed.

## configPath implementation

XDG_CONFIG_HOME checked manually first, then `os.UserConfigDir()`, then `$HOME/.config` fallback if UserConfigDir errors. Decision from prior session: macOS `os.UserConfigDir()` ignores XDG_CONFIG_HOME entirely, so the manual check is required for cross-platform correctness.

## yaml.v3 leniency

yaml.v3 silently accepts constructs you'd expect to fail — unclosed brackets (`[unclosed`), certain malformed sequences. The reliable way to generate a parse error is tab-indented content (YAML explicitly forbids tabs). Discovered because the invalid.yaml fixture used `[unclosed bracket` which yaml.v3 parsed without error.

Compounding issue: a fixture extension typo (`.yml` vs `.yaml`) looked identical to a missing-file case — both return `&Config{}, nil` with no error. The missing-file contract can silently mask test bugs like this.

## Packages don't log, callers do

Decision: no logging inside `Load`. Three options were considered — `slog.Default()`, accepting a `*slog.Logger` parameter, or letting the caller log. Go convention is option 3: packages communicate through return values, callers decide what's worth logging. The cobra root command is the right place to log "no config file found" if needed.

## t.Setenv over os.Setenv in tests

`t.Setenv("KEY", "value")` automatically restores the original value when the test ends — no `t.Cleanup` needed. `os.Setenv` requires manual cleanup and will leak into other tests if forgotten.
