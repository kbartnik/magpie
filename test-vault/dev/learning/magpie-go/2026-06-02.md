## Cobra: PersistentPreRunE Propagation

`PersistentPreRunE` on the root command runs before every subcommand in the tree ‚Äî not just the root. This is why vault resolution lives there: `magpie inbox`, `magpie archive`, etc. all get a resolved vault automatically without any per-command wiring.

Critical gotcha: **if a subcommand defines its own `PersistentPreRunE`, it silently overrides the parent's** ‚Äî the root hook won't run. The fix is to call the parent explicitly from the subcommand's hook. This is a well-known cobra trap.

## Cobra: RunE vs Run

`Run` is void ‚Äî you own all error printing and exit codes. `RunE` returns an error ‚Äî cobra prints it and exits with code 1. `RunE` is almost always correct because:

- Error handling is consistent across all commands without repetition
- Tests can inspect the returned error instead of capturing stderr
- Combined with `SilenceUsage: true`, cobra won't dump usage text on every non-usage error

## Config Path: XDG vs Cross-Platform

XDG Base Directory spec is Linux-native. On macOS the native convention is `~/Library/Application Support/`; on Windows it's `%APPDATA%`. Go's `os.UserConfigDir()` abstracts all three ‚Äî it returns `$XDG_CONFIG_HOME` on Linux, the Library path on macOS, and AppData on Windows.

Decision: use `os.UserConfigDir()` as the platform-correct base. However, `os.UserConfigDir()` on macOS does NOT honor `XDG_CONFIG_HOME` ‚Äî it ignores the env var entirely. To support power users on any platform who've set `XDG_CONFIG_HOME`, check it manually first:

```go
func configPath() string {
    if xdg := os.Getenv("XDG_CONFIG_HOME"); xdg != "" {
        return filepath.Join(xdg, "magpie", "config.yaml")
    }
    base, err := os.UserConfigDir()
    if err != nil {
        base = filepath.Join(os.Getenv("HOME"), ".config")
    }
    return filepath.Join(base, "magpie", "config.yaml")
}
```

Test implication: the "unset" case should compute expected value using `os.UserConfigDir()` in the test itself ‚Äî don't hardcode `~/.config` or it breaks on macOS.
