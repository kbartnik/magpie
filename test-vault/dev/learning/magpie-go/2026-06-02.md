## Cobra: PersistentPreRunE Propagation

`PersistentPreRunE` on the root command runs before every subcommand in the tree — not just the root. This is why vault resolution lives there: `magpie inbox`, `magpie archive`, etc. all get a resolved vault automatically without any per-command wiring.

Critical gotcha: **if a subcommand defines its own `PersistentPreRunE`, it silently overrides the parent's** — the root hook won't run. The fix is to call the parent explicitly from the subcommand's hook. This is a well-known cobra trap.

## Cobra: RunE vs Run

`Run` is void — you own all error printing and exit codes. `RunE` returns an error — cobra prints it and exits with code 1. `RunE` is almost always correct because:

- Error handling is consistent across all commands without repetition
- Tests can inspect the returned error instead of capturing stderr
- Combined with `SilenceUsage: true`, cobra won't dump usage text on every non-usage error

## Config Path: XDG vs Cross-Platform

XDG Base Directory spec is Linux-native. On macOS the native convention is `~/Library/Application Support/`; on Windows it's `%APPDATA%`. Go's `os.UserConfigDir()` abstracts all three — it returns `$XDG_CONFIG_HOME` on Linux, the Library path on macOS, and AppData on Windows.

Decision: use `os.UserConfigDir()` as the platform-correct base. However, `os.UserConfigDir()` on macOS does NOT honor `XDG_CONFIG_HOME` — it ignores the env var entirely. To support power users on any platform who've set `XDG_CONFIG_HOME`, check it manually first:

```go
func configPath() string {
    if xdg := os.Getenv("XDG_CONFIG_HOME"); xdg != "" {
        return filepath.Join(xdg, "magpie", "config.yaml")
    }
    base, err := os.UserConfigDir()
    if err != nil {
        base = filepath.Join(os.Getenv("HOME"), ".config")
    }
    return filepath.Join(base, "magpie", "config.yaml")
}
```

Test implication: the "unset" case should compute expected value using `os.UserConfigDir()` in the test itself — don't hardcode `~/.config` or it breaks on macOS.

## Go Testing — t.Skip and t.Run grouping

`t.Skip("not yet implemented")` is the standard TDD stub pattern — marks tests as work-in-progress without breaking the suite. `t.SkipNow()` is the same without a message; `testing.Short()` pairs with `-test.short` for skipping slow tests.

The non-table `t.Run` pattern groups related cases under one `Test*` function using prose names instead of a struct slice. Each subtest gets its own pass/fail and can be targeted with `-run TestLoad/missing`. This is what the config test stubs use.

```go
func TestLoad(t *testing.T) {
    t.Run("valid YAML file returns correct fields", func(t *testing.T) {
        t.Skip("not yet implemented")
    })
    t.Run("missing file returns empty Config, no error", func(t *testing.T) {
        t.Skip("not yet implemented")
    })
}
```

## testify

Added `github.com/stretchr/testify v1.11.1` to `go.mod`. Lands as `// indirect` until a test file imports it; promotes to direct after `go mod tidy`.

**assert vs require:** identical APIs — `assert` continues on failure (like `t.Errorf`), `require` stops immediately (like `t.Fatalf`). Rule: `require` for setup/preconditions, `assert` for independent value checks.

**Common assertions:**

```go
assert.Equal(t, want, got)
assert.Nil(t, err) / assert.NotNil(t, cfg)
assert.True(t, ok) / assert.False(t, cfg.Verbose)
assert.Len(t, results, 3)
assert.Empty(t, list) / assert.NotEmpty(t, list)
assert.Contains(t, str, "substr")      // strings, slices, maps
assert.ElementsMatch(t, want, got)     // same elements, any order
```

**Error assertions:**

```go
require.NoError(t, err)                           // most common setup guard
assert.Error(t, err)
assert.ErrorIs(t, err, fs.ErrNotExist)            // unwraps chain
assert.ErrorAs(t, err, &pathErr)
assert.EqualError(t, err, "config: missing url")  // exact message
```

`require.NoError` is the reflex after any setup call — if it errors, the rest of the test is meaningless.

**Message suffix:** every assertion accepts a printf-style message via the `f` suffix:

```go
assert.Equalf(t, want, got, "case %q: unexpected result", tc.name)
```

## testify/suite

`testify/suite` provides xUnit-style setup/teardown. Struct fields hold shared state; lifecycle methods initialise and clean up.

```go
type LoadSuite struct {
    suite.Suite
    cfgPath string
}

func (s *LoadSuite) SetupSuite()   { /* once before all tests */ }
func (s *LoadSuite) SetupTest()    { /* before each test method */ }
func (s *LoadSuite) TearDownTest() { /* after each test method, always */ }

func (s *LoadSuite) TestValidFile() {
    cfg, err := config.Load(s.cfgPath)
    s.Require().NoError(err)
    s.Equal("https://example.com", cfg.URL)
}

func TestLoad(t *testing.T) { suite.Run(t, new(LoadSuite)) }
```

Inside suite methods: use `s.Require()` / `s.Assert()`, or call assertions directly on `s`. The `*testing.T` is `s.T()`.

| Hook | Runs |
|---|---|
| `SetupSuite` / `TearDownSuite` | once for the whole suite |
| `SetupTest` / `TearDownTest` | before/after each test method |

**Suite vs table-driven:** they solve different problems. Table-driven = same function, many inputs (data varies, assertions are identical). Suite = shared infrastructure, many functions (struct holds state, each method tests something different). They compose — a suite method can contain a table-driven loop using `s.Run` instead of `t.Run`.

Reach for suite when you're copy-pasting setup across test functions. For one or two tests with a one-line setup, `t.Cleanup` is lighter.

## runtime.Caller in test helpers

`runtime.Caller(skip)` returns the file path of the call stack frame `skip` levels up — `0` = the function containing the call, `1` = its caller, etc. The file path is absolute and baked in at compile time.

In a single-package test helper, `skip=0` anchors paths to the helper file itself. In a **shared** helper (e.g. `internal/testhelpers/`) intended for use across packages, use `skip=1` so paths resolve relative to the *calling* test file — i.e. whichever package is using the helper — not the helper's own directory.

```go
// shared helper — resolves testdata/ relative to the caller's package
func Fixture(name string) string {
    _, file, _, _ := runtime.Caller(1)  // 1 = caller of this function
    return filepath.Join(filepath.Dir(file), "testdata", name)
}
```

This makes `Fixture("valid.yaml")` always find `<caller-package>/testdata/valid.yaml` regardless of where the helper lives.
